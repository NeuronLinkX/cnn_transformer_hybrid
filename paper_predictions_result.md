본 문서는 [paper.md](/home/jiwoo/Desktop/workspace/cnn_transformer_cifar10/paper.md)의 연구 방향을 실제 실행 결과에 연결하여, `runs/dgx_spark_full_cpp/cpp_full_test/predictions.tsv`와 `runs/dgx_spark_full_cpp/cpp_full_test/cpp_infer.log`에 기록된 C++ 배치 추론 결과를 논문형 해설로 정리한 것이다. 핵심 목적은 단순히 "정확도가 얼마였는가"를 나열하는 데 있지 않다. 오히려 CNN과 Transformer를 결합한 이유가 무엇이며, 그 결합이 어떤 종류의 오류를 줄이고 어떤 종류의 표현 학습을 가능하게 하는지, 그리고 현재 결과가 그 가설을 어느 정도 지지하는지를 구조적으로 설명하는 데 초점을 둔다.

## 1. 연구 맥락과 분석 목적

`paper.md`에서 제시한 연구의 핵심 문제의식은 분명하다. CNN은 지역적 패턴, 경계, 텍스처, 형태 조각을 안정적으로 추출하는 데 강하지만, 복잡한 장거리 상호작용이나 이미지 전역의 문맥을 직접적으로 모델링하는 데는 구조적 한계가 있다. 반대로 Transformer는 self-attention을 통해 전역 문맥을 모델링할 수 있으나, 소규모 데이터셋에서는 CNN이 가진 강한 귀납 편향이 없기 때문에 학습이 비효율적이거나 불안정해질 수 있다. CIFAR-10처럼 클래스 수는 적지만 시각적 유사성이 높은 객체들이 공존하는 데이터셋에서는 이 두 특성이 직접 충돌한다.

예를 들어 `cat`과 `dog`, `automobile`과 `truck`, `airplane`과 `ship`은 저수준 패턴 일부를 공유한다. 이런 상황에서는 단순히 지역 텍스처만 잘 잡는다고 충분하지 않으며, 동시에 전역적 윤곽과 배치 관계만 본다고 해결되는 것도 아니다. 따라서 본 프로젝트가 택한 전략은 "CNN으로 먼저 안정적인 지역 표현을 만들고, Transformer로 그 표현들 사이의 관계를 다시 읽게 하자"는 순차적 결합이다.

이 보고서는 다음 질문에 답한다.

1. 현재 하이브리드 모델은 CIFAR-10 테스트셋에서 실제로 어떤 수준의 성능을 보였는가?
2. 오류의 분포는 어떤 구조를 가지며, 그것이 하이브리드 구조의 장점과 한계를 어떻게 드러내는가?
3. CNN 단독, ViT 단독과 비교할 때 하이브리드 구조가 왜 합리적인 선택인지 현재 결과로부터 무엇을 말할 수 있는가?
4. Python 학습 경로와 C++ 배포 경로가 얼마나 일관된가?

## 2. 실험 대상 모델과 추론 경로

현재 분석 대상은 `python/models/hybrid.py`에 구현된 `CNNTransformerHybrid`이다. 이 모델은 다음 구조를 가진다.

- 입력: `224 x 224`로 리사이즈된 CIFAR-10 이미지
- CNN 백본: ImageNet 사전학습 `ResNet-18`
- CNN 출력: `B x 512 x 7 x 7`
- 토큰화: `49`개 공간 토큰으로 펼친 뒤 `Linear Projection`
- Transformer: `embed_dim=512`, `depth=2`, `num_heads=8`
- 분류 방식: `[CLS]` 토큰 기반 최종 선형 분류

즉, 이 모델은 원본 이미지를 Transformer가 처음부터 패치로 읽는 ViT 단독 구조가 아니다. 또한 마지막에 attention을 덧붙인 단순 보조 모듈도 아니다. CNN이 먼저 `7x7`의 밀도 높은 feature map을 만들고, Transformer는 이 49개 공간 표현 간의 관계를 재조합한다. 따라서 이 구조의 핵심 가치는 "지역 특징의 안정성"과 "전역 관계 모델링"의 분업에 있다.

실제 배포 경로는 다음과 같다.

`best.pth -> hybrid_cifar10_full_ts.pt -> build_dgx_spark/cifar10_infer -> predictions.tsv / cpp_infer.log`

이 점은 매우 중요하다. 본 결과는 단순 Python 테스트가 아니라, TorchScript를 거쳐 LibTorch C++ 바이너리에서 재현된 결과이기 때문이다. 따라서 학술적 의미뿐 아니라 배포 적합성까지 함께 평가할 수 있다.

- Macro Recall: `0.9479`

클래스별 샘플 수가 CIFAR-10에서 균등하기 때문에 weighted F1도 거의 동일하다. 이는 모델이 일부 클래스에만 편중되어 전체 accuracy를 끌어올린 것이 아니라, 전반적으로 균형 잡힌 분류 성능을 보인다는 의미다.

## 4. 학습 과정이 보여주는 하이브리드 구조의 실질적 효과

이번 결과를 해석할 때 가장 먼저 주목해야 할 부분은 학습 로그이다.

- Phase 1 최고 검증 정확도: `0.9056`
- Phase 2 최고 검증 정확도: `0.9546`
- Python 테스트 정확도: `0.9489`
- C++ 테스트 정확도: `0.9479`

여기서 Phase 1은 CNN 백본을 고정하고 Transformer 및 분류 헤드 중심으로 학습한 구간이고, Phase 2는 전체 모델을 end-to-end로 미세조정한 구간이다. 이 차이는 단순한 학습 스케줄 차원이 아니라 구조 해석상 중요한 메시지를 담고 있다.

첫째, 사전학습 ResNet-18 백본 위에 Transformer 기반 상단 구조만 얹은 상태에서도 검증 정확도 `90.56%`를 달성했다. 이는 CNN 백본이 제공하는 feature representation이 충분히 강력하며, Transformer가 그 위에서 의미 있는 시퀀스 수준 재조합을 수행하고 있음을 시사한다.

둘째, 이후 전체 네트워크를 함께 미세조정하자 검증 정확도가 `95.46%`까지 상승했다. 절대값으로 `+4.90%p` 향상이다. 이는 단순히 "백본이 좋아서"가 아니라, CNN 표현 자체가 Transformer가 요구하는 토큰 구조에 더 잘 맞도록 재적응되었음을 의미한다. 즉, 하이브리드의 효과는 두 모듈을 병렬로 쌓아두는 데서 나오지 않고, joint fine-tuning을 통해 서로의 표현 공간을 맞춰가는 과정에서 발생한다.

셋째, Python 테스트 `94.89%`와 C++ 테스트 `94.79%`의 차이는 `0.10%p`에 불과하다. 이는 TorchScript 변환과 LibTorch 추론으로 이어지는 배포 경로가 원본 Python 모델을 매우 충실하게 재현하고 있음을 보여준다. 연구용 성능이 배포 단계에서 무너지지 않았다는 점은 실용적 가치가 높다.

## 5. 클래스별 결과가 말해주는 것

클래스별 recall을 보면 다음과 같은 특징이 드러난다.

- 높은 성능 그룹: `ship 0.980`, `frog 0.974`, `horse 0.974`, `automobile 0.964`, `airplane 0.963`, `truck 0.960`
- 중간 성능 그룹: `deer 0.952`, `bird 0.934`
- 낮은 성능 그룹: `dog 0.899`, `cat 0.879`

이 분포는 우연적이지 않다. 오히려 하이브리드 구조가 어떤 문제에서는 강하고 어떤 문제에서는 여전히 어려움을 겪는지 잘 드러낸다.

### 5.1 상대적으로 잘 맞는 클래스

`ship`, `airplane`, `truck`, `automobile`은 형태적 차이가 비교적 뚜렷하고, 객체 외형이나 배경 단서가 비교적 안정적이다. 예를 들어 `ship`은 수평선이나 물 배경과 함께 나타날 가능성이 높고, `airplane`은 하늘 배경과 길쭉한 동체 구조를 갖는다. `frog` 역시 질감과 형태가 비교적 분명하다.

이러한 클래스에서는 CNN이 지역 패턴을 안정적으로 추출하고, Transformer가 객체의 전역 배치와 윤곽 관계를 통합하면서 높은 성능을 낸 것으로 해석할 수 있다. 다시 말해, 지역 특징과 전역 구조가 서로 보강되는 전형적인 성공 사례다.

### 5.2 어려운 클래스: cat와 dog

가장 두드러진 실패는 `cat -> dog 65건`, `dog -> cat 65건`이다. 클래스별 recall도 `cat 0.879`, `dog 0.899`로 가장 낮다.

이 결과는 매우 중요하다. 왜냐하면 이것이 바로 하이브리드 구조가 해결하려던 문제의 중심이기 때문이다. 고양이와 개는 부분 텍스처, 얼굴 방향, 털 색상, 귀와 눈의 배치 등에서 상당한 중첩을 가진다. CNN은 이런 유사한 지역 패턴에서 혼동할 수 있고, Transformer는 전역 관계를 보강할 수 있지만 CIFAR-10 원본 해상도가 매우 작기 때문에 고수준 구조 차이를 충분히 복원하기 어렵다.

즉, 현재 결과는 하이브리드가 모든 시각적 모호성을 제거하지는 못했음을 보여준다. 그러나 동시에 이 실패는 왜 하이브리드가 필요한지도 역설적으로 설명한다. 단순 CNN만으로는 부분 패턴 유사성에 더 취약할 가능성이 크고, ViT 단독은 이 작은 데이터 환경에서 안정적인 구조 학습이 더 어렵다. 현재 하이브리드 모델은 이미 `95%` 근방까지 올라왔지만, 가장 마지막에 남는 오류는 "지역적으로 닮고 전역적으로도 소형 저해상도에서는 쉽게 구분되지 않는 클래스"에 집중되어 있다.

### 5.3 구조적으로 가까운 클래스 쌍

오류 상위 방향은 다음과 같다.

- `truck -> automobile`: `25`
- `automobile -> truck`: `22`
- `airplane -> ship`: `23`
- `ship -> airplane`: `12`
- `deer -> horse`: `21`
- `bird -> deer`: `18`
- `bird -> cat`: `18`

이들은 모두 시각적 구조 혹은 배경 맥락이 일부 겹치는 클래스들이다.

- `truck`과 `automobile`은 바퀴, 차체, 측면 실루엣을 공유한다.
- `airplane`과 `ship`은 파란 배경, 수평 구조, 길쭉한 주체라는 공통점이 있다.
- `deer`와 `horse`는 네 발 달린 동물이라는 형태적 유사성이 강하다.

이러한 패턴은 모델이 무작위로 틀리는 것이 아니라, 의미론적으로 가까운 클래스 사이에서 실수하고 있음을 보여준다. 이는 보통 representation이 완전히 무너졌을 때가 아니라, 상당히 잘 학습되었으나 경계 클래스에서 decision boundary가 촘촘해진 경우에 나타나는 현상이다.

## 6. high-confidence wrong prediction이 의미하는 바

이번 결과에서 특히 주목해야 할 부분은 오분류 평균 confidence가 `0.8388`로 여전히 높다는 점이다. 더 나아가 `Most Confident Wrong Predictions`에는 confidence가 `0.9998 ~ 1.0000`인 오답이 다수 존재한다.

예시는 다음과 같다.

- `ship -> airplane`, confidence `1.0000`
- `frog -> bird`, confidence `1.0000`
- `automobile -> truck`, confidence `1.0000`
- `cat -> dog`, confidence `1.0000`
- `deer -> horse`, confidence `1.0000`

이것은 단순히 모델이 "헷갈렸다"는 뜻이 아니다. 오히려 모델이 내부 표현 공간에서 해당 샘플을 잘못된 클래스 쪽으로 매우 강하게 밀어 넣었다는 뜻이다. 즉, 일부 오류는 불확실성의 문제가 아니라 representation bias 혹은 calibration mismatch의 문제다.

학술적으로 이는 두 가지 의미를 가진다.

첫째, 하이브리드 구조는 전반적 정확도는 높지만 confidence calibration은 별도 보정 여지가 있다. Temperature scaling, label smoothing, mixup/cutmix, focal loss 등의 기법이 후속 연구로 자연스럽게 연결된다.

둘째, attention이 있다고 해서 항상 더 "조심스러운" 예측을 하는 것은 아니다. Transformer는 전역 문맥을 통합하지만, 잘못된 문맥 단서를 확신 있게 결합할 수도 있다. 예컨대 배경, 객체 방향, 윤곽 일부가 특정 클래스와 강하게 연결되면 높은 confidence의 systematic error가 발생할 수 있다.

## 7. lowest-confidence prediction이 의미하는 바

가장 낮은 confidence 샘플들을 보면 `0.3499`에서 `0.5046` 사이에 분포하며, 이 안에는 정답도 오답도 함께 존재한다.

예를 들어:

- `frog -> cat`, `0.3499`
- `cat -> airplane`, `0.3908`
- `airplane -> airplane`, `0.4081`
- `bird -> bird`, `0.4118`
- `cat -> cat`, `0.4191`

이는 decision boundary 근처에 있는 샘플들이며, 모델이 내부적으로 클래스 간 경쟁을 끝까지 유지한 경우다. 중요한 점은 이런 샘플이 오히려 "정상적인 불확실성"을 보여준다는 것이다. 반대로 진짜 위험한 오류는 confidence가 매우 높은 오답이다.

따라서 배포 관점에서는 단순 accuracy 외에도 confidence threshold 기반 재검토 정책이 유용할 수 있다. 다만 현 결과에서는 low-confidence만 걸러서는 충분하지 않다. high-confidence wrong sample이 존재하기 때문이다.

## 8. CNN 단독 대비 하이브리드의 효과를 어떻게 해석할 것인가

이 질문에는 엄밀한 부분과 해석적 부분을 분리해서 답해야 한다.

### 8.1 엄밀하게 말할 수 있는 것

현재 저장소에는 동일 설정에서 학습된 `ResNet-18 단독`과 `ViT-Small 단독`의 실제 결과 파일이 존재하지 않는다. 따라서 "하이브리드가 CNN 단독보다 몇 %p 높다" 혹은 "ViT 단독보다 어느 정도 우수하다"는 수치를 이 보고서에서 단정적으로 쓸 수는 없다. 그런 주장은 추가 실험이 필요하다.

즉, 본 보고서가 직접적으로 증명하는 것은 다음뿐이다.

- 제안된 하이브리드 모델이 CIFAR-10 C++ 배포 경로에서 `94.79%` 정확도를 달성했다.
- 2단계 fine-tuning을 통해 `90.56% -> 95.46%` 검증 정확도 향상이 있었다.
- 오류는 의미론적으로 인접한 클래스에 집중되어 있으며, 전반적으로 균형 잡힌 confusion structure를 보인다.

### 8.2 그럼에도 강하게 시사하는 것

비록 직접 비교 실험 수치는 없지만, 현재 구조와 결과는 하이브리드 설계의 필요성을 강하게 뒷받침한다.

첫째, 순수 CNN은 지역 패턴에 매우 강하지만, `airplane/ship`, `truck/automobile`, `deer/horse`처럼 전역 실루엣과 부위 관계가 중요한 구분에서는 한계가 생기기 쉽다. 현재 하이브리드 모델이 이런 클래스들에서 `0.95+` 수준 recall을 보인 것은 Transformer가 전역 구조 통합에 실제로 기여했을 가능성을 시사한다.

둘째, 순수 ViT는 CIFAR-10처럼 작은 데이터셋에서 inductive bias 부족으로 인해 학습 효율이 떨어질 수 있다. 반면 본 구조는 ResNet-18의 사전학습 표현을 기반으로 Transformer가 작동한다. 즉, Transformer가 처음부터 원시 픽셀 패치를 해석하지 않아도 되기 때문에 sample efficiency 측면에서 훨씬 유리하다.

셋째, 가장 어려운 클래스가 `cat/dog`에 집중된다는 것은 모델이 이미 쉬운 저수준 패턴 분리는 대부분 해결했음을 의미한다. 만약 표현 학습이 약했다면 confusion matrix는 훨씬 더 광범위하게 무너졌을 것이다. 현재의 오류 패턴은 "CNN 지역 특성과 Transformer 전역 문맥이 상당 부분 성공적으로 결합되었지만, 미세한 유사 종간 분리는 여전히 어려운 상태"로 읽는 것이 합리적이다.

## 9. ViT 단독 대비 하이브리드의 효과를 왜 기대하는가

`paper.md`의 문제 설정에 비추어 보면, 본 연구가 순수 ViT가 아니라 하이브리드를 택한 이유는 단순 성능 욕심이 아니라 데이터 조건 때문이다.

CIFAR-10의 원본 해상도는 `32x32`이고, 본 프로젝트는 이를 `224x224`로 업샘플링해 사용한다. 그러나 업샘플링은 정보량 자체를 늘리지는 않는다. 원래 작은 이미지의 제한된 세부정보를 더 큰 입력 공간에 재배치할 뿐이다. 따라서 처음부터 패치 단위로 이미지를 읽는 ViT는 원본 정보가 빈약한 환경에서 불리할 수 있다.

반면 CNN 백본은 locality bias를 이용해 적은 정보에서도 비교적 안정적인 edge, texture, shape primitive를 추출한다. 그런 뒤 Transformer가 이 feature token들 간 장거리 관계를 읽으면, ViT가 처음부터 raw patch 관계를 배우는 것보다 더 효율적인 표현 학습이 가능해진다.

쉽게 말해:

- ViT 단독은 "전역 문맥은 잘 보지만, 작은 데이터에서 무엇을 먼저 봐야 하는지 덜 알려져 있다."
- CNN 단독은 "무엇을 먼저 봐야 하는지는 잘 알지만, 멀리 떨어진 단서들을 유연하게 연결하는 능력은 제한적이다."
- 하이브리드는 "CNN이 먼저 안정적인 시각 어휘를 만들고, Transformer가 그 어휘들 간 문법을 학습한다."

현재 결과는 이 설계 철학이 적어도 실용적으로 매우 설득력 있음을 보여준다.

## 10. C++ 배포 결과가 갖는 추가 의미

이번 보고서는 Python 평가만으로 끝나지 않는다. `hybrid_cifar10_full_ts.pt`를 통해 TorchScript로 내보낸 후 LibTorch C++ 배치 추론에서 `94.79%`를 재현했다는 사실이 중요하다.

이것은 세 가지 의미를 가진다.

첫째, 연구용 모델이 배포용 런타임으로 이전될 때 성능 붕괴가 크지 않다.

둘째, `.pth`와 `.pt`의 역할 분담이 명확하다.

- `.pth`: 학습 재개, 평가, 재실험을 위한 Python 체크포인트
- `.pt`: C++ 런타임에서 직접 로드 가능한 배포 아티팩트

셋째, Python 테스트 `94.89%`와 C++ 테스트 `94.79%` 사이의 `0.10%p` 차이는 오히려 연구적으로 유의미하다. 이는 전처리 세부 구현, floating-point 수치 차이, TorchScript trace 경로 등의 영향이 배포 성능에 미세한 차이를 줄 수 있음을 보여준다. 실서비스 관점에서는 이런 차이를 보고 "배포 경로 검증이 실제로 필요하다"는 결론을 내릴 수 있다.

## 11. 한계와 해석상의 주의점

본 보고서가 매우 강한 정성적 해석을 제공하더라도, 다음 한계는 분명히 남아 있다.

1. 동일 조건의 `resnet18` 및 `vit_small` 베이스라인 결과가 현재 저장소에 없다.
2. 따라서 하이브리드의 우월성을 절대 수치로 직접 입증하지는 못한다.
3. calibration, robustness, inference latency에 대한 비교 실험도 아직 없다.
4. 고신뢰 오답이 존재하므로 accuracy가 높더라도 확률 출력의 신뢰성을 그대로 받아들이기는 어렵다.

즉, 본 보고서는 "하이브리드 구조가 왜 타당한가"를 강하게 설명하지만, "베이스라인보다 반드시 몇 % 더 좋다"는 최종 결론을 완성하려면 추가 실험이 필요하다.

## 12. 결론

현재 C++ 배치 추론 결과는 제안된 CNN-Transformer 하이브리드 구조가 CIFAR-10에서 충분히 경쟁력 있고, 배포 가능한 수준의 성능을 가진다는 점을 분명하게 보여준다. 정확도 `94.79%`, macro F1 `94.78%`, 균형 잡힌 confusion matrix, 그리고 Python-to-C++ 경로의 작은 성능 차이는 이 모델이 단순한 연구용 시연을 넘어 실질적인 엔지니어링 가치를 가진다는 근거가 된다.

무엇보다 중요한 것은 오류의 성격이다. 모델은 무작위로 실패하지 않는다. `cat/dog`, `airplane/ship`, `truck/automobile`, `deer/horse`처럼 시각적으로 인접한 경계 클래스에서 집중적으로 실패한다. 이는 하이브리드 모델이 이미 대부분의 쉬운 분리를 해결했으며, 남은 문제는 미세한 의미 경계의 정교화라는 것을 뜻한다.

따라서 본 결과는 다음과 같이 요약할 수 있다.

- CNN과 Transformer의 결합은 CIFAR-10 같은 소규모 이미지 분류에서 충분히 설득력 있는 전략이다.
- CNN은 안정적인 지역 표현을 제공하고, Transformer는 그 표현들 사이의 전역 관계를 정제한다.
- 2단계 fine-tuning은 이 결합을 실제 성능 향상으로 연결하는 핵심 메커니즘이다.
- 배포 경로에서도 성능이 유지되므로 연구 결과의 실용성이 높다.
- 다만 단독 CNN/ViT 대비 정량적 우위는 후속 베이스라인 실험으로 반드시 보강되어야 한다.

## 13. 후속 실험 제안

논문 완성도를 높이려면 다음 실험이 바로 이어져야 한다.

1. 동일 설정(`image_size=224`, `batch_size=64`, `epochs=30`)에서 `resnet18`, `vit_small`, `hybrid`를 모두 학습하여 정량 비교표를 작성한다.
2. accuracy뿐 아니라 macro F1, class-wise recall, confusion matrix를 함께 비교한다.
3. Python 추론과 C++ 추론의 latency 및 accuracy 차이를 모델별로 비교한다.
4. high-confidence wrong sample에 대해 temperature scaling을 적용해 calibration 개선 효과를 측정한다.
5. `cat/dog`와 `automobile/truck` 오류를 줄이기 위해 mixup, cutmix, label smoothing, stronger augmentation을 추가 실험한다.

이 실험들이 추가되면 `paper.md`의 연구 가설, 구현, 배포, 정량 검증이 하나의 완성된 논문 구조로 자연스럽게 연결될 수 있다.
