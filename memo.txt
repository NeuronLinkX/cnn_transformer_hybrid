 1. 전체 시스템 흐름

  1. 사용자가 train.py를 실행하면 YAML 설정을 읽고, 시드를 고정하고, 장치를 선택한 뒤, CIFAR-10 DataLoader 3개를 만듭니다.
  2. DataLoader는 원본 CIFAR-10 32x32 이미지를 모두 224x224로 키운 뒤 정규화해서 모델에 넣습니다.
  3. 모델이 hybrid이면 ResNet-18 백본 + Transformer encoder + 선형 분류기 구조를 만들고, resnet18 또는 vit_small이면 베이스라
     인 모델만 만듭니다.
  4. 하이브리드 모델은 2단계 학습입니다. 1단계는 CNN 백본을 얼리고 Transformer와 head만 학습합니다. 2단계는 전체를 풀어 end-
     to-end 미세조정합니다.
  5. 각 epoch마다 학습 루프를 돌고, 검증셋 정확도가 최고면 best_phase1.pth, best_phase2.pth, 그리고 대표 파일 best.pth를 저장
     합니다.
  6. 학습이 끝나면 best.pth를 다시 읽어 테스트셋을 평가하고, 마지막 상태를 final_hybrid.pth로도 저장합니다.
  7. 이후 export_torchscript.py가 같은 모델 구조를 다시 만들고 best.pth의 model_state_dict를 넣은 뒤 torch.jit.trace()로 .pt
     파일을 생성합니다.
  8. C++ 바이너리는 이 .pt 파일을 읽고, OpenCV로 이미지를 읽어 Python과 같은 RGB 변환, 리사이즈, 정규화를 적용한 뒤 LibTorch
     forward()를 호출해 예측합니다.

  2. 데이터 파이프라인을 흐름도로 풀면

  1. 입력 데이터 소스는 torchvision.datasets.CIFAR10입니다.
  2. 학습셋은 두 벌로 생성됩니다. 하나는 증강용 full_train_aug, 하나는 검증용 full_train_eval입니다.
  3. 둘 다 같은 원본 학습셋을 보지만 transform만 다릅니다.
  4. 학습 transform 순서는 Resize(224,224) -> RandomCrop(224, padding=16) -> RandomHorizontalFlip -> ToTensor ->
     Normalize(mean,std) 입니다.
  5. 검증/테스트 transform 순서는 Resize(224,224) -> ToTensor -> Normalize(mean,std) 입니다.
  6. 분할은 50,000장의 학습셋을 랜덤 시드 고정 후 90% train / 10% val로 나눕니다. 즉 실제 학습은 45,000, 검증은 5,000입니다.
  7. 테스트셋은 원본 CIFAR-10 test 10,000장을 그대로 씁니다.
  8. DataLoader 수준에서는 train_loader만 shuffle=True, 나머지는 False입니다.
  9. CUDA가 있으면 pin_memory=True, num_workers > 0이면 persistent_workers=True입니다.
  10. 따라서 흐름도에는 “원본 CIFAR-10 -> 학습용 증강 경로 / 검증용 비증강 경로 / 테스트용 비증강 경로”로 갈라지는 분기 노드를
     두면 정확합니다.

  3. 하이브리드 모델 내부를 레이어 단위로 풀면

  1. 입력 텐서 크기는 B x 3 x 224 x 224입니다.
  2. 백본은 timm.create_model("resnet18", num_classes=0, global_pool="")로 만듭니다. 즉 최종 분류기는 제거하고 공간 feature
     map만 남깁니다.
  3. ResNet-18 백본 출력은 설계상 B x 512 x 7 x 7입니다.
  4. 이 7x7 공간 격자 49개가 Transformer에 들어갈 토큰의 원재료입니다.
  5. feat.flatten(2).transpose(1,2)로 바꾸면 B x 49 x 512가 됩니다. 여기서 각 토큰은 “한 공간 위치의 512채널 특징 벡터”입니다.
  6. self.proj 선형층이 각 토큰을 512 -> embed_dim으로 사영합니다. 현재 설정 파일에서는 embed_dim=512라서 차원이 우연히 같지
     만, 코드상으로는 독립 모듈입니다.
  7. 학습 가능한 CLS token 하나를 배치 크기만큼 복제해 앞에 붙입니다. 그러면 시퀀스 길이는 49 + 1 = 50입니다.
  8. 학습 가능한 위치 임베딩 pos_embed를 더합니다. 길이는 고정 1 + 49로 잡혀 있습니다.
  9. Dropout 후 TransformerEncoder로 보냅니다. 현재 기본 설정은 depth=2, num_heads=8, mlp_ratio=4.0, activation=gelu,
     batch_first=True, norm_first=True 입니다.
  10. Transformer 출력에서 첫 번째 토큰, 즉 CLS 위치만 꺼냅니다.
  11. LayerNorm을 통과시킨 뒤 최종 Linear(embed_dim, 10) head가 CIFAR-10 10개 클래스 로짓을 만듭니다.
  12. 따라서 모델 흐름도는 Image -> ResNet18 feature extractor -> 49 spatial tokens -> Linear projection -> CLS prepend +
     Positional embedding -> Transformer encoder x2 -> CLS extraction -> LayerNorm -> Classifier입니다.
  13. 중요한 제약도 하나 있습니다. 코드가 49개 공간 토큰을 강하게 가정합니다. 이미지 크기나 백본을 바꾸면 python/models/
     hybrid.py 의 positional embedding 길이와 C++의 224x224 전처리를 같이 바꿔야 합니다.

  4. 학습 루프를 실행 순서대로 풀면

  1. parse_args()가 --config, --model, --device, --output-dir 등을 받습니다.
  2. load_config()가 YAML을 읽고, CLI override가 있으면 batch_size, num_workers를 덮어씁니다.
  3. set_seed()가 Python, NumPy, PyTorch, CUDA 시드를 모두 고정합니다.
  4. get_device("auto")는 cuda -> mps -> cpu 순으로 장치를 결정합니다.
  5. build_model()이 설정에 따라 하이브리드 또는 베이스라인을 생성합니다.
  6. 하이브리드면 freeze_backbone(True)를 호출해 백본 파라미터의 requires_grad를 끕니다.
  7. 이때 train() 메서드도 오버라이드되어 있어서, phase1 동안 model.train()이 호출돼도 백본만큼은 eval() 상태를 유지합니다. 즉
     BatchNorm running stat까지 고정하려는 의도입니다.
  8. train_phase()는 CrossEntropyLoss, AdamW, CosineAnnealingLR를 생성합니다.
  9. optimizer는 filter(lambda p: p.requires_grad, model.parameters())를 써서 현재 학습 가능한 파라미터만 받습니다.
  10. run_epoch()는 배치마다 forward -> loss -> backward -> grad clip 1.0 -> optimizer.step() 순서로 실행합니다.
  11. epoch 종료 후 evaluate()가 검증셋에서 loss, acc, confusion_matrix, macro_f1를 계산합니다.
  12. 검증 정확도가 최고면 체크포인트에 model_name, model_state_dict, config, epoch, phase, val_acc를 함께 저장합니다.
  13. phase1 종료 후 freeze_backbone(False)로 전체 네트워크를 풀고 phase2를 같은 방식으로 다시 돕니다.
  14. 마지막에는 best.pth를 다시 로드해서 테스트셋을 평가하므로, 최종 테스트는 “마지막 epoch 모델”이 아니라 “검증 최고 모델”
     기준입니다.

  5. TorchScript export와 C++ 추론 흐름

  1. export_torchscript.py는 학습 때와 같은 모델 구조를 pretrained=False로 다시 생성합니다.
  2. 체크포인트에서 model_state_dict만 꺼내 load한 뒤 eval() 모드로 전환합니다.
  3. 1 x 3 x 224 x 224 더미 입력으로 torch.jit.trace()를 수행합니다.
  4. 저장 결과가 .pt이며, 이것이 C++ 배포용 계약 파일입니다.
  5. C++의 preprocess()는 imread(BGR) -> BGR2RGB -> resize 224x224 -> float32 /255 -> HWC to NCHW -> Normalize(mean,std)를 적
     용합니다.
  6. mean/std 값도 Python과 동일하게 하드코딩되어 있습니다. 이 부분이 Python과 C++ 사이 입력 계약의 핵심입니다.
  7. model.forward({input})의 출력 로짓에 softmax를 씌워 확률을 구하고, argmax가 예측 클래스가 됩니다.
  8. 단일 이미지 모드에서는 결과를 이미지에 덧그려 저장합니다.
  9. manifest 배치 모드에서는 manifest.tsv를 읽고 한 줄씩 예측해 predictions.tsv를 씁니다.
  10. 배치 모드에서는 정답 라벨이 있으면 C++ 쪽에서도 confusion matrix와 accuracy를 계산해 stdout에 출력합니다.

  6. 실제 흐름도용 최종 문장

  1. 사용자 명령이 시작점입니다.
  2. 설정 로드와 시드/장치 결정이 첫 제어 노드입니다.
  3. 그 다음 데이터 준비와 모델 생성이 병렬 개념의 준비 단계입니다.
  4. 모델이 hybrid인지 여부가 첫 번째 분기입니다.
  5. hybrid=yes면 phase1(backbone freeze) -> validation -> best checkpoint -> phase2(unfreeze) -> validation -> best
     checkpoint 흐름입니다.
  6. hybrid=no면 단일 학습 단계만 거칩니다.
  7. 이후 두 경로는 best.pth reload -> test evaluation -> final checkpoint 저장으로 합쳐집니다.
  8. 배포 흐름은 best.pth -> 모델 재구성 -> TorchScript trace -> .pt 저장 -> C++ 로드 -> OpenCV 전처리 -> LibTorch forward ->
     softmax/argmax -> 결과 저장입니다.
  9. 데이터 관점에서는 32x32 CIFAR 원본 -> 224x224 정규화 텐서 -> CNN 공간 특징 -> 49개 토큰 -> CLS 추가된 50개 시퀀스 ->
     Transformer -> 10개 로짓이 핵심 선형 흐름입니다.
